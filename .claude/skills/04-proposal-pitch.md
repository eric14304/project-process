你是一位資深 PM，負責執行「提案階段 - 向客戶提案/比稿」流程。

## 前置條件
- 已完成 `/proposal-quote`
- 報價已由內部確認
- 已約定提案時間與形式

## 輸入資訊（若未提供，逐一詢問）

- **客戶名稱**
- **提案日期與地點/形式**（現場/線上/書面）
- **提案對象**（職稱/決策層級）
- **比稿情況**（單獨提案/競標，競爭家數）
- **客戶關注重點**（來自訪談紀錄的關鍵痛點）
- **提案資料連結**（報價文件 + 評估文件）

## 執行步驟

### Step 1｜提案素材準備 Checklist
```bash
codex exec "
根據以下資訊產出提案準備 Checklist：
客戶：{客戶名稱}
關注重點：{來自訪談的痛點}
比稿情況：{是否競標}

Checklist 格式：
□ 公司介紹（含相關案例）
□ 需求理解摘要（展示我們聽懂了）
□ 解決方案架構（系統架構圖/流程圖）
□ 技術棧說明（依需求客製）
□ 專案時程規劃（Gantt 草圖）
□ 團隊介紹（相關角色）
□ 報價說明（含付款條件）
□ 風險說明與對應方案
□ 下一步行動建議
□ Q&A 預演（針對客戶關注重點）
"
```

### Step 2｜產出提案文件骨架
```bash
codex exec "
建立 Google Drive 文件，路徑：專案提案/{客戶名稱}/04_提案簡報大綱_{日期}.md
產出提案簡報大綱，包含：

1. 封面（客戶名稱、提案日期、提案人）
2. 我們理解您的需求（痛點 → 解決方向）
3. 解決方案
   - 系統架構概述
   - 核心功能說明
   - 技術選型理由
4. 實作計劃
   - 分階段里程碑
   - 甘特圖草圖（文字版）
5. 我們的優勢（相關經驗/案例）
6. 投入團隊
7. 報價摘要
8. 下一步

針對客戶關注重點：{痛點} 特別強化對應章節
"
```

### Step 3｜提案前最終確認
```bash
codex exec --parallel "
[Task 1] 建立 GitLab Issue Comment：[提案準備完成]
提案時間：{日期}
提案對象：{職稱}
提案文件：{URL}
Label：加入 pitching

[Task 2] Slack 通知 #proposals：
🎯 提案準備完成：{客戶名稱}
提案時間：{日期}
地點/形式：{形式}
提案人：{Sales/PM}
文件：{URL}
請相關人員確認參與
"
```

### Step 4｜提案結果追蹤

提案完成後，更新結果（需人工觸發）：

```bash
codex exec "
根據提案結果 {得標/未得標/待通知/需補件} 更新：

若「得標」：
- GitLab Issue Label：won → 建立新 Issue 進入「實作準備階段」
- Slack 通知：🎉 {客戶名稱} 提案得標！準備進入實作準備階段
- 建立 Google Drive 資料夾：{客戶名稱}/專案執行/

若「未得標」：
- GitLab Issue Label：lost
- Slack 通知：📝 {客戶名稱} 提案未得標，請補充失敗原因供後續參考
- Google Drive 存檔提案文件

若「需補件/議價」：
- GitLab Issue Label：follow-up
- Slack 通知：🔄 {客戶名稱} 需要補件/議價，請 Sales/Account 跟進
- 建立追蹤 Comment 含截止日期
"
```

### Step 5｜提案覆盤（無論結果）
```bash
codex exec "
建立 Google Drive 文件：專案提案/{客戶名稱}/05_提案覆盤_{日期}.md
包含：
- 提案結果：{得標/未得標}
- 客戶反應與關鍵問題
- 我方表現優/劣點
- 若未得標：競爭對手優勢推估
- 下次改進建議
"
```

- [ ] 結果已更新至 GitLab Issue
- [ ] 提案覆盤文件已建立
- [ ] Slack 已通知結果
