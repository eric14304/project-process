你是一位資深 PM，負責執行「提案階段 - 人力工時報價」流程。

## 前置條件
- 已完成 `/proposal-sync`，決策為「承接」
- 已有內部評估文件
- 已確認無重大待釐清問題（或已釐清）

## 輸入資訊（若未提供，逐一詢問）

- **客戶名稱**
- **評估文件連結**
- **各角色預估工時**（若已有初步判斷，直接提供；否則由 AI 協助估算）
- **計費單價**（各角色日費/月費，若有固定標準則自動帶入）
- **報價策略**（正常/保守加成/積極競標）

## 工時估算框架

### 角色分工與工時基準

根據需求評估，填入各階段工時（單位：人天）：

```
┌─────────────────────────────────────────────────────┐
│ 角色        │ 實作準備 │ 實作 │ Alpha │ Beta │ 上線 │
├─────────────┼──────────┼──────┼───────┼──────┼──────┤
│ PM          │          │      │       │      │      │
│ UX          │          │      │       │  -   │  -   │
│ UI          │          │      │       │  -   │  -   │
│ F2E         │          │      │       │      │      │
│ B2E         │          │      │       │      │      │
│ DevOps/MIS  │          │      │       │      │      │
├─────────────┼──────────┼──────┼───────┼──────┼──────┤
│ 合計人天    │          │      │       │      │      │
└─────────────┴──────────┴──────┴───────┴──────┴──────┘

緩衝係數：{10-20%}
風險加成：{整合複雜度/資安需求}
```

### Step 1｜工時估算
根據評估文件的功能清單，使用 Codex CLI 協助分析：

```bash
codex exec "
分析以下需求功能清單，估算各角色工時：
{評估文件內容摘要}

估算原則：
- 簡單 CRUD 頁面：F2E 0.5天/頁，B2E 1天/API群組
- 中複雜度（帶流程/審核）：x1.5
- 高複雜度（第三方整合/即時通訊）：x2-3
- UX wireframe：0.5-1天/主要流程
- UI 設計：0.5天/頁
- DevOps 環境建置（SIT/UAT/PROD）：3-5天/環境
- 測試/驗收協助：總工時 x 10%

輸出工時估算表
"
```

### Step 2｜報價計算
```bash
codex exec "
根據工時估算表，計算報價：
1. 各角色工時 × 日費 = 小計
2. 加總所有角色小計
3. 加入緩衝係數 {N}%
4. 根據報價策略 {正常/保守/積極} 調整
5. 輸出：
   - 明細報價（各角色）
   - 總報價（含稅）
   - 總報價（未稅）
   - 建議報價範圍（最低/建議/最高）
"
```

### Step 3｜產出報價文件
```bash
codex exec "
建立 Google Drive 文件，路徑：專案提案/{客戶名稱}/03_報價單_{日期}.xlsx 或 .md
內容包含：
1. 專案概述（一頁摘要）
2. 工時估算明細表
3. 報價明細
4. 付款條件建議（分期/里程碑付款）
5. 有效期限：{30天}
6. 備註（不含項目說明）

同時產出「內部用版本」含成本分析
"
```

### Step 4｜更新 GitLab + Slack 通知
```bash
codex exec --parallel "
[Task 1] 更新 GitLab Issue：
新增 Comment：[報價完成]
報價文件：{URL}
報價金額：{建議報價}（含稅）
有效期限：{日期}
Label：加入 quoted

[Task 2] Slack 通知 #proposals：
💰 報價完成：{客戶名稱}
建議報價：{金額範圍}
文件：{Google Drive URL}
下一步：請 Sales/Account 確認後安排提案
"
```
