# 實作準備階段 - 人力甘特圖

## 已知資訊（來自真實案例）

### 工具
- **MS Project（.mpp）**：百靈佳案使用，可精確計算工時、假日排除、依賴關係
- **Excel / Google Sheets**：替代方案，適合無 MS Project 授權時
- 粒度通常以**週**為單位；特殊模組可單獨建一份細部時程表

### 版本更新時機
- **重大範疇變更時**更新版本號（V1 → V2 → ...）
- 百靈佳案共產出 V1~V5：合約簽訂 → 規模確認 → 追加 AorB 模組 → 再調整
- 加入新模組後文件名加上說明（例：`V4(含A+B)`）
- 每次更新重新提供給客戶，同步通知上線日期是否調整

### 命名規範
```
{客戶縮寫}_{專案名稱}_執行時程表V{n}_{說明}_{YYYYMMDD}.mpp
例：BI會員集點_執行時程表V4(含A+B)_20230825.mpp
```

## 執行步驟

1. 取得 SOW 功能清單與各角色工時估算
2. 確認合約起始日、各里程碑驗收節點
3. 用 MS Project 或 Excel 排入各角色進場順序
4. 標示里程碑（Design 確認 / SIT / UAT / PROD 上線）
5. 預留客戶審核等待時間（wireframe 過稿 ~1 週、UI 過稿 ~1 週）
6. 輸出 PDF + 原始檔，Kickoff 時提供客戶

### 範疇變更時
- 確認新增功能工時
- 更新版本號（Vn+1）、文件名加上變更說明
- 重新對齊上線日期，通知客戶
