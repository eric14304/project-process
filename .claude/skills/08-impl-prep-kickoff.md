## 待補充問題（逐一確認後完善此 SKILL）

### 關於會議形式
- Kickoff 是現場還是線上為主？
- 通常時長多久（1hr/2hr）？
- 是否需要錄影存檔？

### 關於出席人員
- 客戶端必要出席：PM/IT/業務/決策者？
- 我方出席：全員還是 PM + 各角色代表？
- 是否需要 Sales/Account 主持？

### 關於會議內容
- 固定議程有哪些？（SOW 確認、溝通機制、工具說明、Q&A）
- 是否需要展示 prototype 或 wireframe（若 UX 已有初稿）？
- 如何確認客戶理解驗收標準？

### 關於後續追蹤
- 會議紀錄由誰整理？格式為何？
- 會議決議事項如何追蹤（GitLab issue / Google Docs action items）？
- 定期進度會議的頻率如何在 Kickoff 時確認？
- **共用進度追蹤表需在 Kickoff 建立並說明使用方式** ✅（見下方說明）

### 關於共用進度追蹤表（Kickoff 必建立）
Kickoff 時應建立一份雙方共用的專案進度表（Google Sheets 或 Excel），追蹤每個頁面單元在各交付階段的狀態。

**欄位結構（以 wireframe → UI → 切版為例）**：
```
單元頁面 | 階段A（wireframe）           | 階段B（UI）                | 階段C（切版）
         | 修改項目 | 確認 | 提供 | 預計 | 修改項目 | 確認 | 提供 | 預計 | 修改項目 | 確認 | 提供 | 預計
```

**狀態標籤**：`已調整` / `已完成` / `待更新` / `調整中` / `先結案`

> 此表由雙方在各階段會議後同步更新（頻率：約每週一版），是「誰欠誰什麼」的唯一來源。
> 階段數與欄位依專案類型調整，不固定為三階段。
