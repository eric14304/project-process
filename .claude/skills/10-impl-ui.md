# 實作進行階段 - UI Design

> 🚧 草稿待完善 — 以下為待確認問題清單

## 待補充問題（逐一確認後完善此 SKILL）

### 關於設計規範
- 是否需要建立 Design System（色彩、字型、元件庫）？
- 客戶是否有品牌規範（Brand Guideline）需遵循？
- RWD 是否在此階段設計（桌機/平板/手機）？

### 關於移交 F2E
- F2E 拿到的設計稿格式：Figma 連結 / Zeplin / 匯出規格？
- 是否需要提供切圖資源（icons/illustrations）？
- UI 與 F2E 如何溝通設計細節（Figma comments / Slack）？

### 關於確認流程
- UI 完成後需要幾輪確認（內部/客戶）？
- 客戶對設計有修改意見時，PM 如何介入評估是否在 scope 內？
- UI 定稿後 F2E 開始實作，UI 還能修改嗎？流程為何？

### 關於圖庫授權
- 使用圖庫的授權規範是什麼（已知國泰產研有圖庫授權需求）？
- AI 生成圖像是否可用？
- Icon 使用哪套（Font Awesome/Material/自製）？
